# APDE 2.0

[![Технологии APDE 2.0](https://skillicons.dev/icons?i=processing,androidstudio,java,kotlin,gradle&theme=light)](https://skillicons.dev)

APDE 2.0 - редактор Processing-скетчей для Android. Он нужен, когда хочется писать и запускать скетчи прямо на телефоне или планшете: открыть проект, поправить `.pde`-файлы, нажать Run и сразу увидеть результат.

Проект еще развивается. Базовый рабочий цикл уже есть, но APDE 2.0 пока не пытается полностью заменить настольный Processing IDE и не экспортирует готовый подписанный APK одним нажатием.

Разработчикам лучше начать с [README.developers.md](README.developers.md).

## Скачать

Файл для установки: [APDE 2.0.apk](./APDE%202.0.apk)

## Что уже умеет приложение

### Работать со скетчами

- Держать проекты во встроенном Sketchbook.
- Создавать, открывать, переименовывать и удалять скетчи.
- Открывать внешние папки и `.pde`-файлы через системный выбор файлов Android.
- Работать с несколькими `.pde`-вкладками в одном проекте.
- Создавать, переименовывать и удалять файлы и папки в выбранной рабочей папке.
- Добавлять файлы в `data/` через экран свойств скетча.

### Помогать писать код

- Подсвечивать Java и Processing-синтаксис.
- Показывать номера строк и нормально прокручивать длинный код.
- Откатывать и возвращать изменения через undo/redo.
- Автоматически закрывать скобки и кавычки.
- Менять размер шрифта и ширину табуляции.
- Форматировать `.pde`-код.
- Искать и заменять текст.
- Подбирать цвета через встроенный RGB/HSB-инструмент.

### Запускать и показывать ошибки

- Собирать скетч на устройстве и запускать его кнопкой Run.
- Показывать ошибки компиляции и ошибки во время работы скетча.
- Выводить в консоль сообщения из `print` и `println`.
- Останавливать запущенное превью из редактора.

## Как начать

1. Открой APDE 2.0.
2. Создай новый скетч или открой папку с существующим проектом.
3. Напиши код в `.pde`-вкладке.
4. Нажми Run.

При первом запуске APDE может открыть системный установщик Sketch Preview. Это отдельное приложение, в котором сейчас запускается превью скетча. Установи его, вернись в редактор и нажми Run еще раз.

Android также может попросить разрешить APDE установку приложений из этого источника. Это связано именно с текущим механизмом превью.

## Как выглядит скетч

Для простого запуска достаточно хотя бы одного непустого `.pde`-файла. Если у скетча есть ресурсы, их можно держать рядом:

```text
MySketch/
  MySketch.pde
  ExtraTab.pde
  data/
  assets/
  code/
  android/
```

При запуске APDE берет `.pde`-вкладки проекта и передает превью поддержанные папки с ресурсами.

## Настройки

В приложении уже есть:

- русский и английский интерфейс;
- темы редактора и интерфейса;
- настройки виброотклика, шрифта редактора, табуляции и консоли;
- экран свойств скетча с именем, package name, версией, SDK-параметрами и ориентацией.

Часть Android-параметров на экране свойств уже подготовлена под будущие сценарии сборки и пока не влияет на все этапы текущего превью.

## Что пока ограничено

- Основной сценарий сейчас - запуск превью, а не экспорт готового Android-приложения.
- Поддержка Android Mode и Processing-проектов еще расширяется.
- `Import Library` и `AI Agent` уже видны в меню инструментов, но пока остаются заглушками.
